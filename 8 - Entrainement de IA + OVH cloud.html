Entrainement de l'IA et utilisation d'OVH cloud ​

L'entraînement du modèle IA nécessite un PC ou un serveur suffisamment puissant, équipé d'une carte GPU. Si vous n'en disposez pas, vous pouvez utiliser un service cloud.

Nous avons utilisé OVH Cloud, qui propose 200 € de crédit gratuit pour tester leurs produits : OVH-cloudOffre OVH

Utiliser les serveurs d'OVH nous permet ainsi d'accéder à des machines suffisamment puissantes. Pour y avoir accéder, il suffit de créer une instance, comprenant un serveur contenant l'équipement (GPU) que l'on souhaite.

Création d'une instance OVH ​

Créer une instance (serveur virtuel) en sélectionnant une configuration incluant un GPU adapté.

Connexion à l'instance ​

Une fois l'instance créée, se connecter via SSH en renseignant une clé SSH dans l'interface OVH.

OVH propose un tutoriel officiel pour cette démarche : Tutoriel vidéo OVH – Connexion SSH

Transfert des fichiers d'enregistrement au serveur ​

Transférer les fichiers d'enregistrement et de calibration (dossier huggingface) vers le serveur.

Pour rappel les fichiers sont contenus dans "C:\Users{votre_utilisateur}.cache\huggingface" sur une machine Windows.

Avec PuTTY installé, utiliser la commande pscp : pscp -i C:\Users\{votre_utilisateur}\{clé_ssh].ppk C:\Users\{votre_utilisateur}\Downloads\{fichier_enregistrement}.zip ubuntu@{IP_serveur}:/home/ubuntu

Syntaxe générique : ​

pscp -i chemin_vers_votre_cle_ssh.ppk chemin_vers_votre_fichier_enregistrement.zip ubuntu@{IP_serveur}:chemin_destination_serveur

Connection au compte huggingface ​

hf login Saisir la clé API disponible dans Profil → Access Tokens sur huggingface.co.

Lancement de l'entrainement ​

lerobot-train--dataset.repo_id={User}/{dataset}--policy.type=act--output_dir=outputs/train/act_{dataset}--job_name=act_so101_test--policy.device=cuda--wandb.enable=false--policy.repo_id={User}/{dataset}

Problème de configuration GPU Nvidia (drivers CUDA) ​

Après le lancement, une erreur peut indiquer que la machine ne dispose pas de cœurs CUDA, même si la carte GPU en est équipée. Cela est dû à une incompatibilité de version des drivers CUDA.

Solution :

  1. Désinstaller les drivers cuda_toolkit existants.
  2. Réinstaller la version correspondant au GPU du serveur.

Syntaxe générique : ​
